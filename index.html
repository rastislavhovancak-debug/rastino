Month Readiness — Checks Explained

🔴 Errors — block invoicing
No active billable assignment
The developer has no assignment to any team that is marked as billable for this month. Go to Developers → edit the developer and check their assignments.
No billing rate
The billing rate (€/hour) is not set for this month. Without a rate, revenue cannot be calculated. Go to Rates and set their hourly rate.
No actuals imported (hours > 0)
No actual timesheet hours imported from SAP, or hours are zero. Invoicing is always based on actuals — forecast is never used. Go to Timesheets → Import.
🟡 Warnings — review recommended
Salary missing
No salary or daily rate entered for this month. Needed to calculate CTC and COGS. Go to Costs → Salaries.
CTC missing
Cost to Center not calculated or entered. Without it, Gross Margin cannot be calculated. Go to Costs → Cost to center and use ⚡ Calc.
Overhead not defined
No overhead set for this location/type for this month. If intentionally zero, set it to 0 explicitly. Go to Costs → Overhead.
🟢 Green — fully ready
All checks passed. Active assignment, billing rate, actual hours, salary, CTC and overhead all defined.
Errors prevent accurate invoicing. Warnings affect Gross Margin accuracy but do not block invoice generation.

Timesheet Checks — Explained

Checks run against the uploaded SAP export. Each row in SAP represents one activity entry for one developer on one day. EU developers are expected to log 8h per working day. India developers are expected to log 9h (or 8h on leave days).
Check 1 — Daily hours
Missing day
A developer appears in the SAP export (has entries in this month) but has no entry at all for a specific working day. Public holidays and weekends are excluded. Contractors are flagged as informational only, not an error.
Wrong hours — EU developer
EU developer logged something other than 8h in regular (non-On-Call) activities on a working day. 0h regular hours is OK if On-Call covers the day. Any other amount triggers this check.
Wrong hours — India developer
India developer logged something other than 9h (normal day) or 8h (leave day) total. Any other total triggers this check.
Leave day issue — India developer
India developer logged exactly 8h total (leave day convention) but some of those hours are on a billable activity (1004). Leave days should be 100% non-billable activities.
Over hours
Developer's total billable hours (activity 1004) for the whole month exceed the maximum working hours defined for their location. Fixed-price WBS rows are excluded from this calculation.
Check 2 — WBS codes
Wrong WBS
Developer logged hours against a WBS code that is not assigned to them in the system. Each developer has a set of allowed WBS codes defined in Developers → assignments. Logging against an unassigned WBS is flagged.
WBS mismatch
There is a mismatch between the activity type and the WBS type: either a billable activity (1004 or On-Call) logged on a non-billable WBS (ending -NB), or a non-billable activity logged on a billable WBS (ending -B).
Check 3 — On-Call
On-Call issue — EU developer
EU developer has an On-Call entry but it is not exactly 16h (the expected On-Call shift). On a public holiday, On-Call must be 24h instead. Any other value is flagged.
On-Call issue — India developer
India developers are not expected to have any On-Call entries. Any On-Call activity or classification found for an India developer is flagged immediately.
Check 4 — Public holidays & weekends
Holiday work
Developer logged regular (non-On-Call) hours on a public holiday defined for their location. Acceptable entries on a public holiday are: no entry at all, or 24h On-Call only. Any other entry is flagged with the holiday name shown.
Weekend work
Developer logged non-On-Call hours on a Saturday or Sunday. On-Call entries on weekends are acceptable. Any regular activity logged on a weekend is flagged.
Contractor — informational
Contractor has a day where billable hours differ from 8h, or is missing a day entry. This is informational only — contractors often have flexible hours. No action required unless the hours look wrong to you.
Activity codes: 1004 = billable T&M | 1007 + "On-Call" classification = EU On-Call | 2xxx/9xxx = non-billable. Timesheet status: 10 = filled | 20 = released | 30 = approved. Issues on approved rows (status 30) are highlighted separately.
